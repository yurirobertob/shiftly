# Growth Agent

Bot deterministic that pushes growth insights to Telegram twice a day.
No LLM in the loop — every insight is generated by a rule in `heuristics.ts`,
filtered by a scoring function in `filter.ts`, and persisted in the `Insight`
table for cooldown / acted-rate tracking.

## Pieces

| File              | Role                                                          |
|-------------------|---------------------------------------------------------------|
| `snapshot.ts`     | Reads DB + Stripe → typed `Snapshot` of current funnel state  |
| `heuristics.ts`   | Library of rules — each returns a `Candidate` if it fires     |
| `filter.ts`       | Senior-PD filter: scores, cooldown, dedupe by category        |
| `runner.ts`       | Orchestrator: snapshot → evaluate → filter → persist → send   |
| `telegram.ts`     | Telegram Bot API client + message formatter                   |
| `types.ts`        | Shared types                                                  |

## API routes

- `POST /api/agents/tick` — runs one cycle. Auth: `Authorization: Bearer ${CRON_SECRET}` or `?key=${CRON_SECRET}`.
- `POST /api/agents/telegram` — webhook for Telegram updates. Auth: `X-Telegram-Bot-Api-Secret-Token: ${TELEGRAM_WEBHOOK_SECRET}`.

## Cron

`vercel.json` schedules two daily runs: `0 12 * * *` (~9h BRT) and `0 21 * * *` (~18h BRT).

## Setup (one time)

1. Apply migration: `npx prisma migrate deploy` (or `dev` locally).
2. In Vercel, set env vars:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_USER_ID`
   - `TELEGRAM_WEBHOOK_SECRET` (any random string)
   - `CRON_SECRET` (any random string)
3. Deploy.
4. Register the webhook (one curl after deploy):
   ```bash
   curl "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setWebhook" \
     -H "Content-Type: application/json" \
     -d '{"url":"https://shiftsly.com/api/agents/telegram","secret_token":"<TELEGRAM_WEBHOOK_SECRET>","allowed_updates":["message"]}'
   ```
5. Optional — set commands:
   ```bash
   curl "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/setMyCommands" \
     -H "Content-Type: application/json" \
     -d '{"commands":[
       {"command":"pensa","description":"Forçar ciclo agora"},
       {"command":"status","description":"Snapshot do funil"},
       {"command":"ultimas","description":"Últimos 5 insights"},
       {"command":"done","description":"Marcar último como feito"},
       {"command":"ruim","description":"Marcar último como ruim"},
       {"command":"skip","description":"Pular último 14d"},
       {"command":"ajuda","description":"Mostrar comandos"}
     ]}'
   ```
6. Smoke test: `curl -X POST "https://shiftsly.com/api/agents/tick?key=${CRON_SECRET}"`.

## Adding a heuristic

Push a new entry to the `heuristics` array in `heuristics.ts`:

```ts
{
  id: "unique-stable-slug",
  category: "acquisition" | "conversion" | ...,
  cooldownDays: 7,
  stages: ["1-10", "10-30"], // optional — restricts to funnel stages
  evaluate(snapshot) {
    if (!conditionFromSnapshot) return null;
    return {
      templateId: "unique-stable-slug",
      category: "acquisition",
      title: "Action-oriented title",
      body: "Markdown-formatted body with concrete next steps",
      evidence: ["data point 1", "data point 2"],
      effort: "S" | "M" | "L",
      impact: "L" | "M" | "H",
    };
  },
}
```

The filter does the rest: cooldown, scoring, category dedupe, top-N cap.

## Telegram commands

- `/pensa` — force one cycle now
- `/status` — current snapshot
- `/ultimas` — last 5 insights
- `/done [id8]` — mark done (last if id omitted)
- `/ruim [id8]` — mark dismissed (last if id omitted)
- `/skip [id8]` — same as ruim, used semantically for "not now"
- `/ajuda` — list commands

`id8` = first 8 chars of the insight id, shown at the bottom of every Telegram message.
